# 💡 Product Philosophy | 제품 철학

> Don't Ask. Understand First.
>
> 묻기 전에 먼저 이해하라.

Users should not be forced to repeatedly input, search, remember, or manage information.

사용자가 반복적으로 입력하고, 검색하고, 기억하도록 강요해서는 안 된다고 생각합니다.

The system should understand context first and allow users to simply confirm decisions.

시스템이 먼저 맥락을 이해하고 사용자는 확인만 하면 되는 경험을 지향합니다.

### Examples

- Why should people remember places? → KeepSpot
- 왜 사람이 장소를 기억해야 할까? → KeepSpot

- Why should people manually manage workflows? → BizFlow
- 왜 사람이 업무 흐름을 직접 관리해야 할까? → BizFlow

- Why should local shopping require complex processes? → NEX24 Mart
- 왜 동네 장보기가 복잡해야 할까? → NEX24 Mart
